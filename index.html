Import into your model
Works with Revit, SketchUp, ArchiCAD, Rhino,
and any software that accepts XYZ point data.
Revit 2022 and later · Toposolid workflow
01
Download your CSV
Click ↓ Download CSV above. The file contains X, Y, Z point data in metres, with a UTM origin coordinate in the filename for georeferencing.
02
Open the Massing & Site tab
In Revit, go to the Massing & Site tab in the ribbon. Make sure you are in a 3D view or a site plan view.
03
Create Toposolid from Import
Click Toposolid → Create from Import → Select Import Instance. In the dialogue, choose Specify Points File and navigate to your downloaded CSV.
04
Set units to Metres
When prompted for units, select Meter. All coordinates in the CSV are in metres relative to the UTM origin shown in the download filename.
05
Terrain created ✓
Revit builds the Toposolid surface from your point cloud. Accuracy depends on your plan — up to 0.4m vertical resolution on Pro with LiDAR data.
💡 Keep point count under 20,000 for best Revit performance. Use 6m or 12m spacing for sites over 1km².
SketchUp Pro · Sandbox tools
01
Download your CSV
Click ↓ Download CSV above. You'll need the Sandbox tools enabled or the free Undet plugin for large point clouds.
02
Enable Sandbox Tools
Go to View → Toolbars → Sandbox. This gives you the From Scratch and From Contours tools in your toolbar.
03
Import the CSV
Go to File → Import, set file type to Text files (*.csv) and select your file. SketchUp places the points as a 3D point cloud.
04
Generate terrain surface
Select all imported points, then use Sandbox → From Contours to generate a triangulated mesh surface.
05
Terrain mesh ready ✓
Your site topography is now a SketchUp group in metres. Use the Tape Measure tool to verify a known dimension if needed.
💡 The Undet plugin handles large Pro CSV point clouds significantly better than the native SketchUp importer.
ArchiCAD 25 and later · Mesh tool
01
Download your CSV
Click ↓ Download CSV above. ArchiCAD accepts XYZ survey data directly via the Mesh tool workflow.
02
Open Site Plan, set units to Metres
Switch to your Site Plan view. Set working units under Options → Project Preferences → Working Units → Metre.
03
Import survey data
Go to File → Interoperability → Merge and select your CSV. Or use Design → Create Mesh from Survey Data if available in your version.
04
Build the mesh surface
Select the imported points and use the Mesh tool to generate the terrain surface. Adjust ridge and valley lines for drainage accuracy.
05
Mesh terrain created ✓
Topography placed at real-world elevation. LiDAR data gives you accurate drainage gradients even on very shallow slopes.
💡 Use the Site Map image (↓ download above) as an underlay in ArchiCAD to visually verify the terrain aligns with your boundary.
Rhino 7/8 · Grasshopper compatible
01
Download your CSV
Click ↓ Download CSV above. Each row is X, Y, Z in metres. No headers to strip — clean plain text ready to import.
02
Import as point cloud
Use the Rhino command _Import and select your CSV. Set delimiter to comma, confirm XYZ column order.
03
Generate surface
Select the point cloud and run _Patch or _Drape for a NURBS surface. In Grasshopper, use a Delaunay Mesh component for a precise TIN.
04
Grasshopper workflow
Read File → Text Split → Construct Point → Delaunay Mesh. Gives you a parametric terrain mesh you can edit live and connect to your BIM model.
05
Terrain surface ready ✓
Export to Revit via Rhino.Inside or save as DWG/DXF. LiDAR accuracy is preserved through the entire pipeline.
💡 Rhino handles very large point clouds well — no need to thin your CSV before importing. Use the full-resolution file.

Why our data is better
for real construction projects
Survey-grade terrain you can import, inspect,
and edit inside your own modelling software.

Most online terrain tools give you a visual, not a buildable model. Topo-grapher is built for the other case. Every output is a clean, editable point set at real-world coordinates, accurate to the resolution of the underlying official survey for your region.

01
Sourced from national mapping authorities
We work with the top-tier LiDAR and photogrammetry datasets published by national mapping agencies and geospatial institutes across Europe, the UK, and North America. The same data your surveyors and planners already reference, made accessible inside the tools you already use.
02
Resolution that actually serves BIM
On Pro, vertical accuracy reaches 0.4m or better in supported regions. That is close enough to drive drainage gradients, tie into site levels, and coordinate with civil engineers without needing to redo the terrain at tender stage.
03
Editable, not baked
The CSV is plain XYZ point data. You import it as a Toposolid, mesh, or NURBS surface and you own it from there. Smooth a point, raise a ridge, trim to a site boundary, the data is yours to shape. No black-box texture maps, no locked formats.
04
Georeferenced, not floating
Every export ships with a UTM origin in the filename. Your model lands at its real-world location, aligned with aerial imagery, cadastral maps, and any other georeferenced layer in your coordination workflow. No guessing, no manual offsets.
05
Consistent across borders
Every region uses a different authority, a different API, a different coordinate system. We handle that so you get a uniform XYZ file whether you are working a site in Copenhagen, the Cotswolds, or California. One workflow, whatever country the project is in.
06
Verifiable against public records
Because the underlying data is from official public sources, every point can be cross-checked against the same datasets used by municipal planners and engineers. If a point looks wrong, you can confirm it against the authoritative record. That matters when you are handing a model to a contractor.
The short version
Other platforms give you a picture of the terrain. We give you the terrain itself, as data, at the resolution your consultants actually need, ready to drop into Revit, Rhino, ArchiCAD, or SketchUp and carry through the whole project.

How it works
Four steps from map to model.
01
Draw your site
Click the polygon tool in the map toolbar, then click around your site boundary on the map. Close the shape with a double-click or by clicking back on the first point. You can draw as many points as you need — curves, odd corners, nothing gets simplified.
💡 Zoom in first for tighter boundaries. Use the satellite layer to trace along actual site edges.
02
Choose point density
Select how close together your terrain points are spaced. 3m and 6m use LiDAR-grade data and are great for construction-level work. 12m is ideal for site feasibility and planning. 20m covers large masterplan territories quickly without massive file sizes.
3m — LiDAR 6m — Precise 12m — Site 20m — Masterplan
03
Set units & distribution
Pick Metres or Feet — the CSV will use whichever you choose. Then pick how points are arranged. Uniform grid gives you a perfectly regular mesh, great for Revit. Natural scatter adds slight variation that helps SketchUp and Rhino generate smoother terrain surfaces on irregular sites.
04
Generate & download
Hit the big Generate terrain button. Processing takes 5–30 seconds depending on site size and density. When it's done, download your CSV and import it straight into Revit, SketchUp, ArchiCAD, or Rhino — see the BIM export guide for step-by-step instructions for each package.
Questions about what point density to use? Drop us a line — we'll tell you exactly what works for your project type.

FAQ
Common questions, contact, and legal info.
What format is the download?
A plain CSV with X, Y, Z columns in metres (or feet if you selected that). No header row — just numbers, ready to import. The UTM origin coordinates are included in the filename so you can georeference your model.
What's the biggest site I can generate?
Free accounts: up to 1 km². Pro: up to 4 km² included, with large-site add-ons up to 100 km² available. Company plans: same as Pro with shared access across your team.
How many sites can I generate per day?
Free: 3 per day. Pro and Company: unlimited. Day limits reset at midnight UTC.
My country isn't listed — will it still work?
Yes. Anywhere not covered by a national dataset falls back to Copernicus GLO-30 (30m global) and then Open-Meteo/SRTM (90m). The tool works globally — just expect lower resolution outside our high-accuracy regions.
Can I use this data for real construction?
The data is sourced from official national surveys and is accurate enough for preliminary design, site feasibility, and BIM coordination. For final engineering decisions, you should verify against official municipal records or commission a land survey.
Does Pro require a contract or minimum term?
No contract, no minimum term. Monthly billing, cancel any time. Company plans are also monthly per seat.
Can I get an invoice / receipts?
Yes. Receipts are sent automatically by Stripe after each charge. For custom invoicing or purchase orders, contact us at aki@topo-grapher.com.
