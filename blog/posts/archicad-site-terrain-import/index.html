How to Import a Site Surface into ArchiCAD

Three practical ways to bring accurate site terrain into ArchiCAD, plus which format gives the cleanest mesh.
21 July 2026 · 4 min · Aki

Importing a site surface into ArchiCAD sounds simple until you try it with real survey data. The Mesh tool accepts a few different inputs, and which one you pick affects mesh density, coordinate accuracy, and how much cleanup you do afterwards.

This post walks through the three import paths ArchiCAD supports for site terrain and when to use each.

What ArchiCAD needs from your terrain data#

The native terrain object in ArchiCAD is the Mesh. It’s a triangulated surface with per-vertex heights, used for placing the building, cut-fill calculations, and renderings.

You can build a Mesh from:

  1. A CSV or TXT file of XYZ points (surveyor’s data)

  2. A DXF or DWG with 3D polylines (contours) or 3D points

  3. An IFC file containing an IfcSite element with terrain geometry

All three work. The one you pick usually comes down to what your data provider gives you.

Method 1: XYZ points from a CSV file#

This is the cleanest path if you have raw elevation data.

  1. Prepare your CSV so each row is X, Y, Z with no header row.

  2. Make sure the units match your ArchiCAD project (usually metres for European projects, feet for US).

  3. In ArchiCAD, go to File → Interoperability → Mesh → Place Mesh from Surveyor’s Data.

  4. Point it at your CSV.

  5. ArchiCAD triangulates the points and drops the Mesh at the origin (or offset, depending on your input coordinates).

Watch out for very large coordinates. If your survey uses full UTM or national grid values (say, six-digit northings), ArchiCAD’s precision suffers when you’re that far from origin. The usual fix is to offset the coordinates so the site sits closer to 0,0 in the CSV, and record the true offset separately for later re-registration.

Method 2: DXF contours#

If your data comes as contour lines instead of points, DXF is the format most CAD tools export cleanly.

  1. Open a fresh worksheet in ArchiCAD.

  2. Go to File → External Content → Merge, then select the DXF.

  3. Set the translator to preserve 3D data. Default translators sometimes flatten Z.

  4. Once the contours are visible, place a Mesh over the area and use Adjust Mesh to Slabs / Surfaces to lift the mesh vertices to the contour elevations.

DXF is slower than direct CSV import and needs more cleanup, but it’s often the format you get from local government terrain services. Contours also give you softer transitions than raw points if the point density is low.

Method 3: IFC site element#

If someone else in the project has already modelled the terrain in another BIM tool (Revit, Rhino, whatever), ask them for an IFC export.

  1. Go to File → Interoperability → IFC → Merge IFC Model.

  2. Choose an IFC translator with IfcSite enabled.

  3. The imported site element comes in as a Morph or Mesh depending on how the sender exported it.

This is the path if you’re joining a project mid-flight and someone else owns the survey. It skips the format conversion step, but the geometry quality depends entirely on how it was authored on the other side.

Which format gives the best result#

For accuracy: XYZ CSV, straight into the Mesh from Surveyor’s Data workflow. No lossy conversion, no over-simplified contours, direct control over point density.

For speed: IFC, if someone else has already done the modelling.

For legacy data: DXF, when contours are all you can get.

Getting the elevation data in the first place#

The tricky part is usually getting a clean XYZ file that matches your project extents and coordinate system. Government elevation data is available for most countries, but it comes in raster formats (GeoTIFF, ASC) that you have to convert to points first, and you need to know the geoid model and CRS to get vertical accuracy right.
