How to Create a Toposolid in Revit From a CSV of Elevation Points

Format your CSV correctly, import it into Revit 2024+, and get a clean Toposolid without the usual coordinate and unit headaches.
23 July 2026 · 4 min · Aki

Revit’s Toposolid (introduced in 2024, replacing the older Toposurface for most workflows) can be built from a CSV of elevation points. The workflow is straightforward once your CSV is formatted correctly, but a few small mistakes in the file will silently break the import.

This post walks through the CSV format Revit expects, the exact steps to import it, and the things that trip people up.

What the CSV needs to look like#

Revit expects each row to be one point, in the form:

X, Y, Z

Rules that matter:

  • Comma separated. Semicolons and tabs will not work. If you’re in a locale where Excel writes semicolons by default (Germany, France, several others), export with a saved CSV format or open the file and swap them for commas before importing.
  • No header row. Revit reads row 1 as the first point. If you have X,Y,Z at the top, Revit tries to import it as a point and either fails or produces a stray vertex at (0,0,0).
  • Dot as decimal separator. Use 123.45, not 123,45. Same locale issue as above.
  • Consistent units. Revit imports the numbers as-is in the units you specify at import. If your project is in metres, the CSV should be metres. If the CSV is in feet and the project is in metres, everything comes in about 3.28 times too small.
  • Reasonable coordinate magnitudes. Revit warns you when points are more than about 20 miles (32 km) from the internal origin. This is not just a warning, geometry precision genuinely degrades that far out. If your XYZ values are full UTM coordinates, subtract a fixed offset before writing the CSV and record the offset for later shared-coordinates setup.

Step by step: CSV to Toposolid#

  1. Open your Revit project. Make sure a site plan view is active.
  2. Go to Massing & Site → Toposolid → Create from Import.
  3. Choose the CSV / points file option.
  4. Browse to your CSV. Set the units in the dialog to match the CSV contents (feet or metres).
  5. Confirm.

Revit reads the points, triangulates them, and places the Toposolid on the site.

Point density: how many is enough#

More points is not always better. A dense CSV (tens of thousands of points from LiDAR) produces a very heavy Toposolid, which slows Revit down and does not render meaningfully better than a moderate-density surface.

Practical guidance:

  • Small residential site (up to about half an acre): 500 to 2,000 points is plenty.
  • Medium commercial site (a few acres): 2,000 to 8,000 points.
  • Large master plan or infrastructure site: 8,000 to 20,000 points, and consider splitting into multiple Toposolids.

If your source data is denser than this, thin it before import. Terrain tools like Topo-grapher let you set an output resolution so you do not have to trim manually.

Coordinate systems and shared coordinates#

If you plan to overlay the Toposolid with other georeferenced data (aerial imagery, GIS layers, infrastructure design), set up shared coordinates before you import.

  • Set your project’s Survey Point to the true site coordinates.
  • Subtract the survey point coordinates from your CSV before export, so the imported points sit near the Revit internal origin.
  • After import, the Toposolid reads correctly in shared coordinates while staying near the internal origin geometrically.

Doing this in the other order (importing full UTM values first, then trying to shift the project) is possible but painful. Set up survey point first.

Common errors and fixes#

“The imported file is not valid.” Nearly always a formatting issue: semicolons instead of commas, an extra header row, or non-numeric values in a row.

Toposolid comes in flat at Z=0. The Z column is being read as text. Check the CSV for stray quotes around numbers.

Toposolid is the wrong scale. Unit mismatch between the CSV and the Revit project. Re-check the import dialog units.

Toposolid has spikes or holes. Duplicate points at the same XY with different Z values create spikes. Points too far apart create holes. Both are fixable by cleaning the CSV before import.

Where the CSV comes from#

You can generate an XYZ CSV from raw survey data, LiDAR, or public elevation datasets. The public datasets (USGS 3DEP, ArcticDEM, Ordnance Survey, national mapping agencies across Europe) are freely licensed for commercial work but come as GeoTIFFs, not CSVs. You have to convert them yourself, or use a tool that does.

Topo-grapher does that conversion in the browser. Pick a site on the map, choose XYZ CSV as the output, and you get a Revit-ready file with the right coordinate handling for your region: geoid correction on the vertical, matching CRS, adjustable point density.
